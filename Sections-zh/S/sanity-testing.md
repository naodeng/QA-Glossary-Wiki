## 关于健全性测试的常见问题？

#### 基础与重要性
- **软件测试中的健全性测试是什么？**
  健全性测试是**[回归测试](/wiki/regression-testing)**的一个子集，侧重于在进行细微更改或 **[Bug](/wiki/bug) 修复**后验证特定功能。它是一种快速、非详尽的检查，旨在确保修改后的特定功能或故障按预期工作。
  与**冒烟测试 (Smoke testing)**（广而浅）不同，健全性测试通常是**窄而深**的，集中在一个或几个功能区域。
  执行健全性测试时，应优先测试那些直接受近期代码更改影响的功能。它通常很简短，能在几小时内完成，对于敏捷开发等快速节奏的环境至关重要。
  在持续集成中，健全性测试在成功的构建和冒烟测试之后触发，充当“把关人”，确保新更改在进行更严格测试前不会破坏核心功能。

- **为什么健全性测试在软件开发生命周期中很重要？**
  它确保近期的更改或 Bug 修复没有对现有功能产生负面影响。它充当代码微调后的**快速健康检查**。这种有针对性的测试通过不重复测试整个应用程序，节省了时间和资源。
  在**敏捷环境**中，健全性测试通常是自动化的，为每次迭代后的应用稳定性提供即时反馈。它有助于维持对软件可靠性的高度信心，特别是在时间或资源受限、无法进行全面回归测试时。

- **健全性测试与冒烟测试有何不同？**
  两者都是验收测试的子集，但目的不同：
  - **冒烟测试**：初步测试，检查新构建的基本功能，确保主要功能正常，构建足够稳定以进行后续测试。问题：“应用能大致运行吗？”（广而浅）
  - **健全性测试**：更具体，在收到带有细微更改的构建后执行，确保特定的问题或功能按预期工作。问题：“近期的更改合理且功能正确吗？”（窄而深）

- **健全性测试的主要好处：**
  - **快速反馈**：即时验证。
  - **成本效益**：专注于特定区域，节省资源。
  - **关注关键问题**：对于发布决策至关重要。
  - **支持持续集成**。

- **健全性测试在敏捷开发中的作用：**
  敏捷团队利用健全性测试确保代码更改没有破坏现有功能。它符合敏捷强调“用户满意度”和“可工作的软件”的原则，通过快速确认更改来维持产品的稳定性。

#### 过程与技术
- **健全性测试过程涉及哪些步骤？**
  1. **识别更改的特性**。
  2. **选择[测试用例](/wiki/test-case)**：覆盖受影响的功能。
  3. **搭建[测试环境](/wiki/test-environment)**。
  4. **执行测试**：手动或自动化。
  5. **分析结果**。
  6. **报告发现**。
  7. **重测 (Retest)**：修复后验证。

- **常用的技术有哪些？**
  - 有选择性的测试用例执行。
  - 基于优先级的测试。
  - [探索性测试](/wiki/exploratory-testing)。
  - 采样测试。
  自动化健全性测试通常集成到 CI/CD 流水线中。

- **健全性测试的典型持续时间：**
  通常很短，从 **15 分钟到几小时**不等。目标是快速平衡彻底性与时间效率。

- **如何在 CI 环境中执行？**
  代码提交 -> 自动构建 -> 自动部署到测试环境 -> 运行预定义的**健全性[测试套件](/wiki/test-suite)** -> 结果分析与反馈。

#### 工具与实践
- **常用工具：**
  - **[Selenium](/wiki/selenium)**：Web 应用自动化。
  - **Appium**：移动端。
  - **Cypress**：现代 Web E2E 测试框架。
  - **[Postman](/wiki/postman)**：API 健全性测试。
  - **TestComplete**, **JUnit**, **NUnit**, **Robot Framework**。

- **如何引入自动化？**
  识别稳定的关键路径，创建自动化测试脚本，并集成到 CI/CD 流水线中。确保测试是**独立的**，以避免级联失败。

- **最佳实践：**
  优先考虑关键路径、维护测试清单、保持测试简单、实施版本控制、及时沟通结果并定期更新测试套件。

- **健全性测试可以复用吗？**
  通常可以**复用**，尤其是当更改是增量式且不影响核心功能时。但如果引入了新特性或重大更改，则需要更新测试。

- **如何记录测试结果？**
  应当简洁明了：
  - **结果 (Outcome)**：通过/失败。
  - **测试的功能**：登录流程、支付网关等。
  - **失败详情**：Bug 描述、重现步骤、截图。
  - **环境详情**。
  - **影响评估与建议**。
