## 关于大猩猩测试的问题？

#### 基础与重要性
- **什么是大猩猩测试？**
**大猩猩测试 (Gorilla Testing)** 是一种**手工测试**技术，涉及对软件应用的特定模块或组件进行高强度且随机的测试。其特点是**重复性**和**攻击性**，测试人员集中于频繁使用某个功能（或一组相关功能），以揭示传统测试方法可能无法发现的缺陷。

- **它是如何进行的？**
测试人员通常会向系统输入各种意外或随机的数据，或者快速执行一系列功能，以确保组件能够承受压力并在极端条件下可靠运行。这包括尝试通过大量输入或快速操作来“搞垮”软件。

- **为什么它很重要？**
它提供了一种针对关键组件的**聚焦测试方法**。通过不断重复测试高风险区域，大猩猩测试有助于发现那些广泛测试策略可能忽略的**难以检测的问题**（如内存泄漏、缓冲区溢出或异常处理）。它还作为其他测试类型的补充，增加了测试深度并提高了对关键功能稳定性的信心。

#### 目标与区别
- **主要目标**：
    *   **验证健壮性**：通过高强度测试验证模块的稳健程度。
    *   **揭示隐藏 Bug**：发现内存、异常处理等深层问题。
    *   **确保可靠性**：通过重复执行任务检查性能的一致性。
    *   **压力测试**：评估特定功能的极限。
    *   **验证容错能力**：故意导致故障并观察系统响应。

- **与其它测试的区别**：
    *   **强度与侧重点**：与覆盖范围广的[功能测试](/glossary/functional-testing)不同，大猩猩测试是针对小而关键片段的“深度打击”。
    *   **手工 vs 自动化**：虽然它也可以部分自动化，但通常更依赖于手工的、详尽的探索，尝试非典型用户行为。
    *   **与[猴子测试](/glossary/monkey-testing)的区别**：猴子测试通常是随机测试整个系统，而大猩猩测试是集中攻击**单个模块**。

#### 实施与最佳实践
- **实施步骤**：
识别关键模块 -> 定义健壮性用例 -> 重复执行测试 -> 改变输入模式 -> 监控系统资源 -> 详细记录与分析 -> 修复并进一步加压测试。

- **常用技术**：
    *   **聚焦攻击 (Focused Attack)**：全方位、全角度攻击特定功能。
    *   **随机测试 (Random Testing)**：在特定模块内进行无预设计划的测试。
    *   **压力测试 (Stress Testing)**：施加重负载以测试稳定性。
    *   **自动化重复执行**：使用脚本循环执行同一操作以检测性能衰减。

- **最佳实践**：
    *   **聚焦核心**：优先测试高风险或历史故障多发模块。
    *   **引入随机性**：模拟不可预测的用户行为。
    *   **短时长高强度**：保持测试人员的专注度。
    *   **迭代反馈**：在修复后立即进行回归测试。

#### 现实应用与挑战
- **应用场景**：金融系统的交易处理、游戏引擎的输入循环（防止动作过快崩溃）、电商的购物车与支付网关（应对促销峰值）、医疗设备的长期运行稳定性、汽车导航系统的重复操作响应。
- **受益行业**：金融、医疗、汽车、航空航天、电信及游戏行业。
- **挑战与局限**：
    *   **范围局限**：过度关注一个模块可能忽略全局或集成问题。
    *   **疲劳性**：重复测试同一功能容易导致漏测。
    *   **资源密集**：对人力和时间要求较高。
    *   **收益递减**：随着主要 Bug 被修复，发现新问题的效率会降低。
